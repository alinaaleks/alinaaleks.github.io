### Что это?

Файл Excel, который создает листы по шаблону, подставляя в них данные.

По нажатию кнопки создается **нужное количество уже заполненных** накладных.

Каждая строка данных автоматически превращается в отдельный заполненный документ.

![Иллюстрация создания накладных](/assets/projects/waybills-creation/macro-working.gif){: width="800" height="auto"}

### Контекст — Зачем?

Упростить и ускорить создание схожих документов.\
Особенно когда их много.

---

### Логика — Как это работает

1. **Подготовка данных.** Нужная информация вносится на листе `data`:

- каждая строка = один документ
- каждый столбец = поле для заполнения в этом документе

![Логика работы создания накладных](/assets/projects/waybills-creation/work-logic.png){: width="800" height="auto"}

Шаг 2. **Шаблон**

- лист `template` определяет структуру ТТН
- данные подтягиваются с листа `data` через формулы (ИНДЕКС + ПОИСКПОЗ)

![Автозаполнение накладной по id](/assets/projects/waybills-creation/work-logic-animated.gif){: width="800" height="auto"}

Шаг 3. **Запуск макроса**

- макрос считает количество строк в данных (= требуемое количество накладных)
- берет шаблон, заполняет, переименовывает, сохраняет на новый лист

Шаг 4. **Готовые накладные**

- каждый документ получает имя `ttn_№`
- система возвращает пользователя на главный экран

### Интерфейс

Инструмент управляется через простое Меню:

- кнопка `Создать ТТН`
- кнопка `Удалить созданные`
- отображение результата выполнения: 1) время последнего запуска, 2) количество созданных файлов

![Главный лист](/assets/projects/waybills-creation/start-sheet.png){: width="600" height="auto"}

### Результат

- автоматическое создание документов за секунды
- устранение ручного копирования
- снижение количества ошибок при заполнении
- возможность повторного использования инструмента

### Сложности и решения

- Сначала в шаблоне использовала формулы с `ВПР`
  → Перешла на `ИНДЕКС + ПОИСКПОЗ`, так как это более гибко в использовании и легче менять структуру таблицы в дальнейшем.
- При запуске макроса сначала удаляются ранее созданные листы, и при первом запуске удалился еще и шаблон (упс 🤭)
  → Потребовалась более устойчивая логика удаления старых файлов, только те, что содержат `ttn_*` в имени
