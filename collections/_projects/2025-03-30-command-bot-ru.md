### Что это?

Telegram-бот - быстрый справочник по командам и горячим клавишам для часто используемых технологий (Git, командная строка, Python, Conda, веб-разработка).

Напоминалка. Позволяет быстро "вспомминать" нужные команды прямо в Telegram через заранее заданную структуру.

Ссылка на бот: [Command Bot](https://t.me/TypeRunBot)

### Контекст - Зачем?

Поймала себя на том, что часто "гуглю" одни и те же команды. Какие-то запоминаются, да, но многие находятся в статусе "Вроде бы помню, но не до конца". Нужно просто напомнить конкретное слово или порядок слов.

Захотелось создать напоминалку, чтобы иметь быстрый доступ к используемым мною командам.

### Цель => Решение

- Собрать часто используемые команды в одном месте.
- Сделать доступ максимально быстрым (через Telegram).
- Просто, быстро, желательно с бесплатным хостингом.
- Возможность быстро вносить правки/добавления.
- Возможность расширяться и увеличивать количество команд.

Реализация через Telebothost.

- нужно ввести название бота и токен (от BotFather)
- команды пишутся на специальном скрипте Tele Bot Lang (похож на JavaScript)

### Логика - Как это работает

Основная команда `/start` выдает список тем/программ.
Темы логически сгруппированы (Git, Python и т.д.).
Структура иерархическая: Программа → Тема → Команды.

Пример: GIT => Init / Delete => Список наиболее используемых команд.

[изображение - логика работы]
(регистрация бота, получение токена - регистрация в telebothost - создание/написание команд - корректировки в реальном времени)

### Сбор и подготовка данных

- Команды собраны из моих заметок
- Организованы локально по структуре: Программа → Тема → Список команд
- Упрощены и адаптированы для вывода в Telegram: без лишнего текста
  Преобразованы под ограничения Telebothost (плоская структура команд)

### Результат

- ускорена работа с программами
- снижено время поиска команд
- удобство использования

### Сложности и решения

Изначально был выбран вариант бот на Python => хранить код на GitHub => откуда он автоматически отправляется в Railway.app.
Это рабочая схема, простая. Но.

1. хостинг платный
2. для локальной разработки я создавала второй бот, чтобы не было конфликтов

Сейчас я перешла на Telebothost.

Плюсы

- быстрый переход к запуску бота (в считанные минуты)
- быстрые корректировки в реальном времени

Минусы

- не унифицировано - чтобы изменить место хостинга нужно будет снова переписывать код. Когда бот на Python - структура и файлы не меняются, может просто измениться площадка для размещения.

- отсутствие контроля версий (нельзя локально на компьютере делать изменения и синхронизировать с хостингом).
  Сейчас я локально веду свою структуру и храню в Github.

В дальнейшем хочу перейти на Oracle Cloud и вернуться к Python, а также самостоятельно управлять размещением на сервере

### Инструменты

- Telegram Bot API / BotFather - регистрация бота, получение уникального токена для работы с Telegram
- Telebothost (TBL scripting)
- Git и GitHub - хранение и управление кодом

Первая версия:

- Python - написание основной логики бота: обработка команд, формирование ответов
- Railway.app - размещение бота в интернете для работы 24/7, запуск приложения на удаленном сервере
