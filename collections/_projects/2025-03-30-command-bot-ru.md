### Что это?

Telegram-бот - справочник по командам и горячим клавишам для часто используемых технологий (Git, командная строка, Python, Conda, веб-разработка).

Напоминалка. Позволяет быстро "вспоминать" нужные команды прямо в Telegram через заранее заданную структуру.

Ссылка на бот: [Command Bot](https://t.me/TypeRunBot)

### Контекст — Зачем?

Поймала себя на том, что часто "гуглю" одни и те же команды. Часто нужно просто вспомнить какое-то конкретное слово или порядок слов.

Поэтому захотелось создать напоминалку, чтобы иметь быстрый доступ к используемым мною командам.

---

### Цель => Решение

- Собрать часто используемые команды в одном месте.
- Сделать доступ максимально быстрым (через Telegram).
- Просто, быстро, желательно с бесплатным хостингом.
- Возможность быстро вносить правки/добавления.
- Возможность расширяться и увеличивать количество команд.

Реализация через Telebothost.

- нужно ввести название бота и его индивидуальный токен
- команды пишутся на специальном скрипте Tele Bot Lang (похож на JavaScript)

![Bot Creation Logic](/assets/projects/command-bot/bot-creation-logic.png){: .center width="600" height="auto"}

### Логика — Как это работает

1. Основная команда `/start` выдает список тем/программ.
2. Темы логически сгруппированы (Git, Python и т.д.).
3. Структура иерархическая: Программа → Категория → Команды.

![Bot Work Logic](/assets/projects/command-bot/bot-work-logic.png){: .center width="600" height="auto"}

<p class="notation">Пример: GIT => DIFF => Список наиболее используемых команд.</p>

### Сбор и подготовка данных

- Команды собраны из моих заметок
- Упрощены и адаптированы для вывода в Telegram: без лишнего текста

### Сложности и решения

Изначально был выбран вариант: бот на Python => код хранится на GitHub => откуда он автоматически отправляется в Railway.app (хостинг) => бот становится активен.
Это рабочая схема, простая. Но.

1. хостинг платный
2. для локальной разработки пришлось создавать второй бот, чтобы не было конфликтов

Telebothost - это решение #2.

**Плюсы**

- быстрый переход от "нета бота" => к "есть бот" (в считанные минуты)
- быстрые корректировки в реальном времени

**Минусы**

- не унифицировано - чтобы изменить место хостинга нужно будет снова переписывать код. Когда бот на Python - структура и файлы не меняются, может просто измениться площадка для размещения.
- отсутствие контроля версий (нельзя локально на компьютере делать изменения и синхронизировать с хостингом).
  Сейчас я локально веду свою структуру и храню в Github.

### На будущее — Идеи для проработки

В дальнейшем хочу перейти на свой сервер и вернуться к Python, чтобы было больше гибкости (а пока нужно найти подходящий и его настроить).

### Инструменты

- Telegram Bot API / BotFather - регистрация бота, получение уникального токена для работы с Telegram
- Telebothost (TBL scripting) - хостинг-платформа специально для Telegram-ботов
- Git и GitHub - хранение и управление кодом

Первая версия:

- Python - написание основной логики бота
- Railway.app - сервис для быстрого запуска приложений, который автоматически обрабатывает код из GitHub
